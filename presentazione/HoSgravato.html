üñ±Ô∏è Deep Dive: Implementazione Drag & Drop

Architettura del Sistema

Il drag & drop utilizza l'API HTML5 nativa con gestione completa degli eventi:

// 1. Inizializzazione del drag
function onDragStart(e) {
  draggedId = this.dataset.id; // Salva ID task
  e.dataTransfer.setData('text/plain', draggedId);
  e.dataTransfer.effectAllowed = 'move';
  this.style.opacity = '0.6'; // Feedback visivo
}

// 2. Durante il trascinamento
function onDragOver(e) {
  e.preventDefault(); // Permette il drop
  e.dataTransfer.dropEffect = 'move';
}

// 3. Entrata in una colonna
function onDragEnter(e) {
  e.preventDefault();
  e.currentTarget.classList.add('drag-over');
}

// 4. Drop finale
function onDrop(e) {
  e.preventDefault();
  const id = e.dataTransfer.getData('text/plain');
  moveTaskTo(id, col.dataset.status);
  col.classList.remove('drag-over');
}

Gestione degli Edge Cases

  • DragLeave intelligente: Controlla le coordinate del mouse per evitare flickering quando si passa sopra elementi figli
  • DragEnd cleanup: Reset dello stato anche se il drop fallisce
  • Variabile globale draggedId: Fallback se dataTransfer non √® disponibile
  • Re-attachment dinamico: Event listeners riattaccati dopo ogni render

üíæ Deep Dive: Sistema di Persistenza

Architettura della Persistenza

Il sistema usa LocalStorage per memorizzare lo stato completo dell'applicazione:

const STORAGE_KEY = 'devtask:data';

// LOAD: Caricamento all'avvio
function load() {
  const raw = localStorage.getItem(STORAGE_KEY);
  if (raw) {
    try {
      tasks = JSON.parse(raw);
      return;
    } catch (e) {
      console.warn('Errore parsing, uso default');
    }
  }
  tasks = defaultData.slice(); // Fallback
}

// SAVE: Salvataggio dopo ogni modifica
function save() {
  localStorage.setItem(STORAGE_KEY,
    JSON.stringify(tasks));
}

Vantaggi della Strategia Scelta

  • Semplicit√†: Nessun backend necessario per iniziare
  • Velocit√†: Accesso istantaneo ai dati (sincrono)
  • Privacy: Dati rimangono sul dispositivo dell'utente
  • Offline-first: Funziona senza connessione internet
  • Zero latency: Nessun ritardo di rete

Gestione Errori e Edge Cases

  • Try-catch: Protegge da localStorage corrotto o non valido
  • Fallback ai default: Array vuoto se load fallisce
  • Validazione implicita: JSON.parse valida automaticamente la struttura
  • Storage quota: LocalStorage ha ~5-10MB (sufficiente per migliaia di task)

Limitazioni Note

  • Dati non condivisi tra browser diversi
  • Possibile perdita se cache viene cancellata
  • No sincronizzazione multi-dispositivo
  • No collaborazione in tempo reale

Queste limitazioni saranno risolte nella roadmap futura con backend integration.

üîç Deep Dive: Sistema di Ricerca

Ricerca Real-Time Multi-Campo

Il sistema di ricerca filtra istantaneamente su tre campi:

searchInput.addEventListener('input', () => render());

// Dentro render()
const q = searchInput.value.trim().toLowerCase();
const filtered = tasks.filter(t => {
  if (!q) return true; // Mostra tutto se vuoto
  return (
    (t.title || '').toLowerCase().includes(q) ||
    (t.desc || '').toLowerCase().includes(q) ||
    (t.assignee || '').toLowerCase().includes(q)
  );
});

Caratteristiche della Ricerca

  • Case-insensitive: Usa toLowerCase() per confronti indipendenti da maiuscole
  • Substring matching: Trova anche corrispondenze parziali
  • Null-safe: Operatore OR per gestire campi vuoti
  • Real-time: Evento 'input' per aggiornamento istantaneo
  • Performance: Filter nativo di JavaScript (O(n) - accettabile per < 1000 task)

UX della Ricerca

  • Placeholder descrittivo: "Cerca per titolo, descrizione o assegn..."
  • Mantiene struttura delle colonne durante filtro
  • Mostra "Nessuna issue" se nessun risultato
  • Contatori mostrano sempre il totale (non il filtrato)
  • Clear visibile tramite X nativo del browser

Possibili Miglioramenti Futuri

  • Ricerca fuzzy (tolleranza errori di battitura)
  • Highlight dei termini cercati nelle card
  • Filtri per priorit√† e stato
  • Ricerca avanzata con operatori (AND, OR, NOT)
  • Salvare ricerche frequenti
